## Challenge Overview

The Splunk Agentic Ops Hackathon invites developers, security engineers, IT engineers, network engineers, and AI-forward builders to create solutions that combine AI with the power of Splunk.

Participants are expected to build intelligent applications that improve how teams:

- Monitor systems smarter
- Detect anomalies earlier
- Investigate incidents faster
- Automate operational and security workflows
- Build better developer experiences on top of Splunk data and APIs

The hackathon emphasizes the use of Splunk's latest AI capabilities, including AI agents for Splunk apps, Splunk MCP Server, Splunk hosted models, Splunk AI Assistant, Splunk AI Toolkit, and AI-powered app development tooling.

---

## What to Build

Build an innovative AI-powered solution that enhances how teams monitor systems, secure environments, or build on the Splunk platform.

The project should leverage one or more Splunk AI capabilities to solve real-world challenges in at least one of these areas:

- Observability
- Security operations
- Network operations
- Developer productivity
- Splunk app development
- AI-assisted operational workflows

The project must be installable, runnable, and consistent with what is shown in the demo video and described in the submission text.

Build solutions that help engineering teams, ITOps teams, or NetOps teams better understand system behavior, detect anomalies earlier, and automate operational responses using AI.

Good project directions include:

Build solutions that enhance the developer experience, automate workflows, or simplify how applications interact with Splunk data and APIs.

Good project directions include:

- SPL query generation or validation tools
- Developer workflow automation

## Recommended Splunk AI Capabilities

Participants are encouraged to use one or more of the following Splunk AI capabilities.

### AI for Splunk Apps

Use the Python SDK to build agentic workflows inside Splunk apps.

Potential uses:

- Custom AI-powered Splunk apps
- Agentic search workflows
- AI-driven alert actions
- Modular input apps
- Custom search app experiences

### Splunk MCP Server

Use the Splunk MCP Server to connect AI agents securely to Splunk data through the Model Context Protocol.

Important note: OAuth support is currently in Controlled Availability and not generally available, so token-based authentication is the recommended way to get started.

Potential uses:

- AI agents that query Splunk data
- Investigation workflows connected to Splunk
- Context-aware operational assistants
- Agent-driven observability or security workflows

### Splunk AI Assistant

Use natural language to generate, edit, explain, or improve SPL queries.

Potential uses:

- SPL query generation
- Query debugging
- Search explanation
- Assistant-driven dashboards or workflows

### Splunk AI Toolkit

Use Splunk AI Toolkit to build custom AI models with operational or security data.

Potential uses:

- Forecasting
- Classification
- Anomaly detection
- Custom insight generation
- Operational ML workflows

### Splunk Hosted Models

Use Splunk-hosted models such as Foundation AI Security Model and Cisco Deep Time Series Model.

Potential uses:

- Security reasoning
- Forecasting
- Time-series analysis
- Natural language understanding
- AI-assisted operational insights

## Submission Requirements

A valid submission must include the following.

### 1. Project

The project must:

- Be built with the required developer tools
- Fit the hackathon theme
- Use or integrate with Splunk capabilities
- Be installable and runnable on the intended platform
- Function consistently with the demo video and written description

### 2. Track Selection

Each submission must identify one track:

- Observability
- Security
- Platform & Developer Experience

### 3. Text Description

The submission must include a text description explaining:

- Project features
- Functionality
- Problem being solved
- Value provided
- How AI is used
- How Splunk is used

### 4. Demo Video

The demo video must:

- Be under **3 minutes**
- Show the project working on its target device or platform
- Demonstrate how AI is used
- Explain the problem being solved
- Highlight the value of the solution
- Be publicly posted on YouTube, Vimeo, or Youku
- Avoid unlicensed third-party trademarks, music, or copyrighted material

Judges are not required to watch beyond the 3-minute mark.

### 5. Public Open-Source Repository

The project repository must be public and open source.

It should include:

- Open-source license file
- All source code
- Required assets
- Clear README documentation
- Setup instructions
- Run instructions
- Required dependencies
- Example configurations or datasets, if applicable

The open-source license should be visible and detectable near the top of the repository page.

### 6. Architecture Diagram

The root of the repository must include an architecture diagram named:

```text
architecture_diagram.md
architecture_diagram.pdf
architecture_diagram.png
```

The diagram should show:

- How the application interacts with Splunk
- How AI models or agents are integrated
- Data flow between services, APIs, and application components

## Judging Process

Judging may happen in multiple rounds and may include expert panels, peer review, automated AI-driven analysis, or a combination of methods.

### Stage One: Baseline Viability

Submissions are evaluated on a pass/fail basis to determine whether the project:

- Fits the hackathon theme
- Reasonably applies the required APIs, SDKs, or Splunk technologies featured in the hackathon

### Stage Two: Scored Evaluation

Submissions that pass Stage One are evaluated using four equally weighted criteria.

---

## Judging Criteria

| Criteria | What Judges Look For |
| --- | --- |
| Technological Implementation | Does the project demonstrate quality software development? |
| Design | Is the user experience and design well thought out? |
| Potential Impact | How big of an impact could the project have? |
| Quality of the Idea | How creative and unique is the project? |

Tie-breaking is handled by comparing tied submissions against the criteria in order. If a tie remains across all criteria, the judges vote on the tied submissions.
